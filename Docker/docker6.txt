when you use docker-compose and docker-compose file in mension different services then to jistbhi services hai with in single docker-compose file vo sab yek "network ka hisa hote hai"

     docker-compose.yml
  different services present  ------------> bydefault all services run on single network


docker-compose  apne aap network bnata hai ,and sari services ko uska part bhi bnata hai ,isa wajhse hme alag se network bnananeki jarurut nhi pdti hai isa case mai

you can see live eg-dono container aapas me kapi aasanise connect ho pa rhe hai

proof

>docker network ls

isa ne aapne aap network bnaya and default name bhi diya

>docker-compose run mypythonapp

-----------------------------------------------------------------------------


docker-compose.yml

services:  
  mysqldb:    
    images: 'Mysql:latest' 

    environment: 
      - MYSQL_ROOT_PASSWORD=root  
      - MYSQL_DATABASE=userinfo
    
    container_name: "mysqldb" 

     network:
        -my-network  //network

    healthcheck:
        test:['CMD' ,'mysqladmin' ,'ping' ,'h' ,'localhost']
        timeout:20s
        retries:10

   mypythonapp:
      build: ./ 
      container_name:myapp

       network:    //isa serives ko bhi network ka part bnate hai
          -mynetwork

      depends_on:
         mysqldb:
           condition: service_healthy
         stdin_open:true
         tty:true


//sabse last mai services upar likha uske level ke sath likna hai ,yo can see in documentation 

networks:
  my-network:
