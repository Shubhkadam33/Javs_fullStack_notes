application.properties-used for external configuration(databases ,port,path,security,jpa) this small file  replace large xml configuration 

Spring boot Architecture:

           client(brower/mobile/API consumer)
                      |
           controller layer(web layer-@Restcontroller/@controller)
                      |
          service layer(Business logic-@service)
                      |
         Respository layer(data access-@respository)
                      |
              (Database (MySQL,postgreSql etc)        

1) for eg  when form submit from browser then data pass to controller using url
 
2)when backend and fronted in separate system that time use @Restcontroller for connect backend and frontend
when both frontend and backend in one place that time use @controller for connect backend and frontend

after receiving data from url then sent to the service file

3)business file-@service 
means In MVC dao file is present for execute query ,here we use @service

In spring 
dao file(only make file of studentDao.java) then write login of query
   but in spring boot there is mandatory to use annotation that is @service
(using this jpa reo we create of obj of that class in this service files so we can access predefined method of hibernate throw jpa ,then query fire then data sent to the database)

4)Respository layer
we create interface file not java file and extend jpa repo
this Jpa(java persistant API) give you readymade method of hibernate like save,update,delete,fetchall,deleteById
   so no need to write write full query only use prefined  method
