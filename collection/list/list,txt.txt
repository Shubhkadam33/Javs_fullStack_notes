

Before Collection Framework (JDK 1.2), we used arrays and legacy classes (vector, stack, dictionary, hashtable, enumeration).
Arrays are fixed in size, have no built-in methods, and are less flexible.

So to overcome these problems, Collection Framework was introduced to provide:

Dynamic size
Type safety (using Generics)
Predefined methods
Better performance
Code reusability

Iterator is interface
It is parent class of collection
It is used for iterate one by one obj/element from collection
Has 3 main methods: hasNext(), next(), remove()

<<<<<<1st eg:>>>>>>>>>>>>

import java.util.*;

public class Demo {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        list.add("A");
        list.add("B");
        list.add("C");

        Iterator<String> itr = list.iterator();

        while(itr.hasNext()) {
            System.out.println(itr.next());
        }
    }
}

📌 Removing element using Iterator (Important for Interview)

Iterator<String> itr = list.iterator();

while(itr.hasNext()) {
    String value = itr.next();
    if(value.equals("B")) {
        itr.remove();   // Safe removal
    }
}
