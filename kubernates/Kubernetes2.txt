In docker case -code bind krte the ,isi tarike se yeha krna pdta hai but tarika alag hai-at this use service obj

>kubectl expose deployment my-nginx --port=80 --type=LoadBalancer

jo port iske under listen kr rha hai vo port dena hai-80 port 
type -yha pe aap alag aalg type de sakte ho but sbse most commonly useful hai ham vo use krege --> "LoadBalancer"

Quick Overview of Commands Used 

minikube start/delete (by using this command we build cluster)
hm minikube use kr rhe hai ,vo single node cluster bnata ho ,jo ki  testing purpose ke liye hota hai so single node or same node mai hi worker node and master node dono work krte hai 
minikube status  (we can check cluster status)
minikube dashboard (we can monitor ,state of cluster ,node ,pods  ,deployment )

kubectl create deployment my-app --image=link (use prefined image and deploy this application) ,image ke bad  name deto ho ,simple agar aap kisi image ka name deto ho vo by default dockerhub se pull kr ta ha  or "kisi dusri registry ko use krna hai aapko proper full path dena pdhega "
kubectl get deployments  (we can check state of deployment) ,jab ham deployment bnate hai ,koi isa images mai problem nhi hoti hai ,kubelet automatic pods bhi bnata hai , kivi ki pod ke under hmara image hai as a form of container run hota hai 
kubectl get pods   ( you can see pods jo bne huve hai)
kubectl delete deployment my-app 

kubetcl expose deployment my-app --type=LoaddBalancer --port=80  (cluster se bahr se app ko access kr sakte hai) ,basically yha pe service bnai thi
minikube service my -app  ( jab aap service bna te ho ,deployments ko assign krate ho ,then uske bad this command use ,so to use that service  ,to aap aapke webapp/service ko access kr sako)
kubectl get services 9if you  want to check your services)
