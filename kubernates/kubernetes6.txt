   name: host-pv

   capacity:
      storage: 1Gi
   VolumeMode:Filesystem
   storageClassName: standard
    accessModes:
       - ReadWriteOnce
    hostPath:
       path: /data/
       type: DirectoryOrCreate

host-pvc.yml 

apiVersion: v1
kind: persistentVolumeClaim
metadata:
  name: host-pvc
spec:
   volumeName : host-pv
   storageClassName:standard
   accessModes:
       -ReadWriteOnce
   resourses:
        requests:
            storage: 1Gi

----------------------------------------- 

metadata:
   name : host-pvc  // claim ka name kya hai
volumeName :host-pv 
//hmara storage bn gya hai usko claim krne vale hai
 
resources kha se le rhe hai --> requests  and storage  (hmara storage bna gya usa maise kit --> at this time pura storage claim krna chah rhe hai so 1Gi yeh request bheji

 documneation --> diff type of persistant vol 
1) container storage interface (CSI)
2) hostpath- hostpath vol ( for single node testing only , will not work in a multi-node cluster ,consider using local volume instead
3)network file system (NFS) storage
4) local - local storage device mounted on nodes  multiple node use kr rhe ho then use "local"

type of volume 
1) emptyDir
this vol similarly to docker hm normal vol bnate the ,docker manage krta tha sa type ka hi hai
  jab bhi pod bnta hai tab yeh dir bn jati hai ,With the pod isko multiple container isko use kr sakte hai
     iska lifecyle -agar pod restart huva or new pod bna so tab data permanently remove hoga

2)hostpath -it is smilar to bind mount jo docker mai use krte the also hmne persistant vala host path dekha tha ,yeh simple host path hai jo persistant vol se thoda difff hai

data management node level pe hoga ,jo bhi pod hai usko use kr sakte hai

this type is not secure kafi chize ke liye recommended nhi krte hai
    kuch cases ke liye useful hai like internal lib use krna ho ,ya temporary file management use krna ho
