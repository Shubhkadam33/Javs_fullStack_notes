Scaling of App k8s

hmne self healing kr liye but there is one problem --> kubertes ne hmare application ko restart kr diya but you notice "kuch second ke liye hmari website down thi "

kuch second down bhi effort nhi kr sakte

but I want app band ho hi na ---> uski liye ham kya kr sakte hai--> dependency hta sakte hai

you can see yek hi pod ,replicate set instance running hai

if we use multiple instance of application then to kya hoga :
uske two benefit
1)single point of failure khatam ho jayega
suppose mnalo yek app ke multiple instance hai,agar yek band ho gya tho dusara to chalu hai
dura band hogya to tisra to chalu hai 
        so hmari webasite hmesha up and running rhegi and no down 2-3 sec

2) agar hare pass traffic jada hai hmare website pe bhut jada traffic aa rha hai ,vha pe load balancing kabhi asani se kr sakte hai

hmara incoming traffic distribute kr sakte hai

uske liye ham kya kr sakte hai --> live env mai application ko scale kr sakte hai
   scale krna means -multiple instances running krna apne application ke

iske liye simple command hai

>kubectl scale deployment node-app --replicas=4
aapko jistne number of instance /replicas dene hai vo de sakte hai
  mri application 4 aalag aalg pod mai chalni chahiye
