package demo;

import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;


@WebServlet("/Addition")
public class Addition extends HttpServlet {
	
	protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
		request.getParameter("num1");
		request.getParameter("num2");
		
		int n1=Integer.parseInt(request.getParameter("num1"));
		int n2=Integer.parseInt(request.getParameter("num2"));
		int result=n1+n2;
		
		System.out.println("addition"+result);
		
	}

}

--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------Different between Get and Post

| **GET Method**                                      | **POST Method**                                      |
| --------------------------------------------------- | ---------------------------------------------------- |
| Data is sent through the **URL**                    | Data is sent inside the **request body**             |
| **Less secure** (data visible in URL)               | **More secure** (data not visible in URL)            |
| Limited data can be sent (URL length limit)         | Large amount of data can be sent                     |
| **Faster** than POST                                | **Slightly slower** than GET                         |
| Used for **fetching** data                          | Used for **sending** or **submitting** data          |
| Can be **bookmarked**                               | Cannot be bookmarked                                 |
| Parameters appear in the browser history            | Parameters are not stored in history                 |
| Used for **idempotent** operations (no data change) | Used for **non-idempotent** operations (data change) |
| Example: `/login?user=abc`                          | Example: form submission (login, signup)             |

Types of JSP Elements or tags

Directives: <%@ page %> – page settings

Scriptlets: <% ... %> – Java code

Expressions: <%= ... %> – output values

Declarations: <%! ... %> – methods/variables


“Expression tag <%= %> is used when we want to display a value on the page.
Scriptlet tag <% %> is used to write Java code, whether single line or multiple lines.”

Directives (<%@ ... %>) – These are used for page-level settings like importing classes or setting error pages.

Scriptlets (<% ... %>) – Here we write Java code inside JSP, but it's not recommended in modern development.

Expressions (<%= ... %>) – Used to directly print or output values on the webpage.

Declarations (<%! ... %>) – Used to declare methods or variables that become part of the servlet class.”**


------------deep explain----------------------------------------
Types of JSP Elements
1. Directives – <%@ ... %>

As it is:
<%@ page %> – page settings

Interview explanation:
“Directives tell the JSP engine how to process the page. They do not generate output.
For example, the page directive is used for page-level settings like import packages, error pages, buffer size, etc.”

2. Scriptlets – <% ... %>

As it is:
Java code

Interview explanation:
“Scriptlets allow us to write Java code directly inside the JSP file.
But using scriptlets is not recommended in modern applications because it mixes Java with HTML, making code hard to maintain.”

3. Expressions – <%= ... %>

As it is:
output values

Interview explanation:
“Expressions are used to directly output a value on the webpage. Whatever we put inside <%= %> is displayed as text in the browser. No need to write out.println().”

4. Declarations – <%! ... %>

As it is:
methods/variables

Interview explanation:
“Declarations are used when we want to define variables or methods that will be part of the servlet class generated from the JSP. These are not executed inside the service method.”
